  字符串处理

   数字转成字符，或者将字符转成数字成为了windows编程常见需求

    字符转数字

     a版

      sscanf_s(buff, "%d", &b);

       buf为接受转换结果的缓冲区

       %d为转换目标格式的格式控制字符

       &b为待转换的变量的空间首地址

      即多字节版 即ascii码版

     w版

      swscanf_s(buffw, L"%d", &b);

      即宽字节版 即unicode码版

     t版

      _stscanf_s(bufft, _T("%d"), &b);

      t版是通用版，可以根据当前程序的编码方式对程序中的字符进行转换

     这里的格式化控制字符%d指的是将字符串转为十进制数

      该参数也可以指定为转为十六进制数和八进制数

    数字转字符

     a版

      sprintf_s(buff,100, "%d",a);

     w版

      swprintf_s(buffw,100, L"%d", a);

     t版

      _stprintf_s(bufft,100, _T("%d"), a);

     函数原型

      int sprintf_s( char *buffer, //字符串指针 size_t sizeOfBuffer, //缓冲区大小 const char *format [, //格式化控制符 argument] ... //变参 );

      int swprintf_s( wchar_t *buffer, size_t sizeOfBuffer, const wchar_t *format [, argument]... );

     sprintf_s函数中的格式化字符%d意思是将数字以十进制的形式写入到字符串中

      这个参数也可以是八进制或十六进制

     注意sprintf()安全版函数需要指定缓冲区大小，但是sscanf_s()函数不需要指定缓冲区的大小

    示例代码

      // 数字转字符 int a = 100; int b = 0; char buff[100]; wchar_t buffw[100]; TCHAR bufft[100]; sprintf_s(buff,100,"%d",a); swprintf_s(buffw,100,L"%d", a); _stprintf_s(bufft,100, _T("%d"), a); //字符转数字 sscanf_s(buff,"%d", &b); swscanf_s(buffw, L"%d", &b); _stscanf_s(bufft, _T("%d"), &b);

    灵活使用sprintf_s函数可以达到字符串拼接的目的

     sprintf_s(buff,100,"分数:%d",a);

     即在格式化字符串处做文章

     bool _trace(TCHAR *format, ...) //变参函数 { TCHAR buffer[1000]; va_list argptr; va_start(argptr, format); //将格式化信息写入指定的缓冲区 wvsprintf(buffer, format, argptr); va_end(argptr); //将缓冲区信息输出 OutputDebugString(buffer); return true; }

     注意该函数拥有可变长参数，调用方法类似于printf("%d",100);其中TCAHR*format指的是格式控制字符，...指的是与格式控制字符相匹配的内容

    可以对其进行改造，使其可以在窗口进行弹窗

      bool NewMessageBox(TCHAR *format, ...) //变参函数 { TCHAR buffer[1000]; va_list argptr; va_start(argptr, format); //将格式化信息写入指定的缓冲区 wvsprintf(buffer, format, argptr); va_end(argptr); //将缓冲区信息输出 //这里是主要的区别，使用弹出窗口的方式显示调试信息 MessageBox(NULL,buffer,NULL,NULL); return true; }

      NewMessageBox((wchar_t *)_T("%d %s"), 1000, L"hello");

    使用这两个函数注意形参为TCHAR* 故格式控制字符需要使用 _T(%d %s)的形式，如果需要将其输出为特定编码格式的内容，需要使用(char*) (wchar_t*)进行强转

   消息的存放

    windows操作系统存在一个系统消息队列

    每一个GUI程序，都有自己的消息队列

    系统消息队列负责将消息分发给不同的程序消息队列，消息在程序消息队列中被消息循环依次进行处理，发给相应的处理函数，产生相应的效果

   消息的处理

    自定义消息的创建和处理

     自定义消息的值只能定义在WM_USER之后，且由于该信息在系统中不存在，所以系统中没有特定的信息触发机制与其对应，所以需要通过系统中已有的消息命令发挥作用

     #define MYMSG WM_USER+1 case WM_LBUTTONDOWN: //鼠标左键点击 { int xPos = LOWORD(lparam); int yPos = HIWORD(lparam); //以非队列的方式向当前窗口发送用户自定义的消息，此时用户的回调函数中需要有MYMSG信息对应的处理过程 //此种发送消息的方式会导致阻塞，只有当该条消息处理完毕后，程序才继续运行 SendMessage(hwnd，MYMSG, 0x1000, 0x500); break; } //例如 case MYMSG: (0,0,0,0);//这里是什么意思？ break;

  队列消息与非队列消息

   系统使用两种方法传递消息

   对列消息发送到消息队列，等待被消息回调函数处理

    Postmessage()

     进入消息队列

     程序不会发生阻塞，会继续执行消息循环

     不存在返回值，仅仅将消息压入队列

     该函数形参与回调函数形参相同

      窗口句柄

      消息码

      附加消息1

      附加消息2

   非队列消息不进入消息队列，直接被消息回调函数处理

    SendMessage()

     不进入消息队列

     程序会发生堵塞，直到消息处理完毕后才会继续执行

     会返回消息回调函数的参数

     该函数形参与回调函数形参相同
