 使用

  包含头文件

   #include<vector>

  声明命名空间

   using namespace vector

  定义的时候需要指定数据类型(vector本身是一个类)

   vector<int>obj;

  支持[]

   及可以使用类似数组的方式对变量量名进行操作

  在插入和删除操作方面效率比较慢

  可以自动扩展空间

  定义: vector<int>vecInt;

  功能

   增加

    vecInt.push_back(0);

     //注意这里是在动态数组最末尾加上一个元素,并且该元素数值为0

     vecInt.push_back(2); vecInt.push_back(3); //这样操作之后,数组内存放的数据为 0,2,3

    vecInt[0]=10;

     将下标为0的位置的元素赋值为10

     此时使用vecInt[1]=10;是错的,因为还没有为下标为一的元素申请内存空间

    插入

     vecInt.insert(vecInt.begin(),4);

      将4插入到数组的首位,其余元素以此向后移

     vecInt.insert(vecInt.begin()+1,5);

      将5插入到数组的数组下标为1的位置,之后的元素以此向后移

   删除

    删除最后一个元素

     vecInt.pop_back();

    删除指定位置的元素

     vecInt.erase(vecInt.begin()+2);

   遍历动态数组

    使用数组的方式遍历

     for(int i=0;i<vecInt.size();i++) { cout<<venInt[i]<<','; } cout<<endl;

    使用c++11标准的新方式进行遍历

     auto根据赋值符号的右边的数据类型来自动推测变量的类型

      例如 auto n=100;此时n为int类型的数据

      auto m;//不赋值的时候无法推测m的数据类型,无法定义变量

      auto &是一个自动的引用类型,主要是为了防止产生拷贝

     for(auto& i : vecInt) { cout<<i<<','; } cout<<endl;

    通用的容器遍历方式:迭代器遍历

     定义一个vector<int>迭代器类型的变量 itr(相当于一个指向vector<int>类型的指针)

      vector<int>::iterator itr;

     for(auto itr=vecInt.begin();itr!=vecInt.end();) { cout<<*itr<<','; if(*itr==5){ itr=vecInt.erase(itr); continue; } ++itr; } cout<<endl;

      // 当一个元素通过迭代器被删除之后 // 这个迭代器就会变成一个无效的迭代器 // 这个无效的迭代器不能使用++,--运算符了 // 如果删除之后还要继续循环 // 就得为迭代器重新赋值 // erase 会删除一个迭代器指向 // 的元素, 同时也会返回一个新的 // 迭代器, 新的迭代器指向被删除 // 元素的下一个位置.

    迭代器的操作

     class MyClass { public: void fun() { cout << "MyClass::fun\n"; } };

      定义一个类

     vector<MyClass> vecMyClass;

      创建一个动态数组,数组名为vecMyClass,数组中的元素均为MyClass类的对象

     vecMyClass.push_back(MyClass());

      为数组添加一个元素

     auto itrMy = vecMyClass.begin();

      使用迭代器指向动态数组的首地址,指向动态数组的第一个元素

     itrMy->fun();

      使用迭代器以指针的方式调用类对象中的成员函数

      // 迭代器的操作 // 迭代器可以看做是一个执行数组元素的指针. // 1. 可以使用自增自减运算符来找到下一个元素 // 2. 可以使用*运算符来取出指针指向的内 // 容/也可以修改指针指向的内容. // 3. 如果容器保存的是类类型的话, 还可以使用 // `->`运算符来访问类类型成员

   访问元素

    cout<<vecInt[0];

    vecInt[1]=vecInt[0];

    vecInt[2]=vecInt.at(0);

     at函数接收一个下标,返回一个元素

  迭代器

   begein()

    可以视为第0个元素的指针

     即*(vecInt.begin()+1)的值为vecInt[1]

   end()

    可以视为指向最后一个元素之后的一个内存空间的指针

     即 *(vecInt.end()-1)的值为动态数组vecInt的最后一个数组元素的值

tips

 空类是一个字节

 虚函数有虚基表占有四个字节内存空间,

  只有一个虚函数的类所占字节要么是1个字节(与空类相同),要么是5个字节,(无论如何,空类中的那一个字节时必须被保存的)

 为什么会产生虚基

  为了解决菱继承二义性的问题

   继承方式与virtual的顺序可以颠倒

 虚继承是为了解决菱形继承,解决二义性问题,不是为了提供多态

 析构函数作为虚函数时可以用于在销毁基类指针的时候顺便调用子类的析构函数

 父类中声名的纯虚函数,子类中不一定非要将其进行实现

 虚函数的机制在父类的与孙子类之间函数的重写也有作用

 重定义的对象也可以是虚函数,即在父类子类中的虚函数也可以为重定义的

 stl中把各种存储数据的结构都称为容器

  stl是通过模板机制实现的

 vector可以储存结构体类型,能够储存任何数据类型

 vector在单纯定义后是没有分配空间的要么进行多次push,要么在定义的时候就在

 使用malloc创建对象的时候不会调用构造函数

  new在申请对象(或多个对象)的同时,会调用相应的构造函数

  new不是函数,是一种运算符

 拷贝构造的形参是一个类型的引用的原因

  因为引用的传参效率高

  因为如果直接用类,可能导致死循环

 为const类型的成员变量初始化可以在初始化列表中进行,也可以在类中定义之后直接给一个数,进行初始化(在vs2015中支持)

 构造的顺序

  父类->成员->子类

  这里如果父类中有对象成员,则在父类构造调用之前,也需要先调用父类中的成员的构造

 浅拷贝是坏处
