 变量种类

  全局变量

   文件作用域中定义

    生存周期在整个程序运行期间

    针对vs而言,文件作用域就是一份源文件

   可以跨文件使用(在同一工程不同文件间调用),只要在其他文件中使用 extern声明该全局变量

   全局变量声明

    extern int g_nNum

     表示该全局变量在其他文件中定义

    extern int g_nNum=0

     这是一个全局变量的定义,而不是声明,

      注意如果向在其他文件中改变全局变量的值需要有两个步骤(前提是该全局变量不是静态全局变量)

       在文件中声明全局变量

       然后改写全局变量的值,使之不等于全局变量定义时候赋的值

    如果想要在本文件(本源代码文件)中使用其他文件的全局变量,需要在本文件使用 extern 要是使用的全局变量进行声明

   有一定命名规范 g_ 开头

    如果一个全局变量和一个局部变量名字相同,使用的时候以就近原则为依据

     int a; int main() { int a; a=0; }

     比如上面的0是赋值给局部变量的

   注意任何变量都只能定义一次,否则会出现变量重复定义报错

  局部变量

   复合语句作用域与函数作用域中定义的

    复合语句或函数运行期间

  静态变量

   static int a=0;

   可以给所有变量或函数前面加这个前缀

   静态属性造成的影响

    对局部变量

     延长局部变量的生存周期,延长到与全局变量相同,程序一开始就进行定义

     但是该变量仍不能在其他作用域中使用

    对全局变量

     生存周期不变

     全局变量如果加了静态属性,则只能在目前的文件使用该全局变量(注意这类全局变量的作用域没有改变,还是本文件)

     对于一个静态全局变量,只能在本文件的范围内使用,如果在其他文件中使用 extern int g_nNum声明全局变量并试图使用,会导致编译器报错

 生存周期

  变量占用内存周期的时间

  一般变量生存周期是从进入作用域开始到离开作用域

 变量的定义一定要在第一次使用之前

 指针变量的大小

  32位下4字节

   包括所有的int* char* double*

   因为指针保存的数据时内存地址,对于32位系统,其内存地址是可以用个四个字节的数据来表示

  64位下8字节

 指针的算数运算

  int *p; p+=1 加一指的是指针指向的内存向下移动4个字节,对于指针变量中储存的数据,为了指向后四个字节,,必须加四个字节的的数值

  相似的 对于double *p; p+=1 ,其实p是中储存的数据时加了8个字节的数

 指针的类型意义在于指针解引用的时候决定需要从指向的地址中取出多少个字节的内容,也决定指针加1的时候,1表示多少个单位
