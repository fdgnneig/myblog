 二维数组的使用

  二维数组的定义

   int array[5][10]={{1,2},{3,4},{5,6}}

  数组的访问

   int array[5][10]={{1,2},{3,4},{5,6}}; *array==array[0];//数组首元素的地址 array //数组的地址 //这里数组首元素的地址与数组的地址值相同，但是如果量地址分别赋值给不同的指针变量，则指针变量做算术运算时增加或减少的值不同，前者自增1表示增加了sizeof(数组元素),后者自增1表示增加了sizeof(数组) array[0]访问第一个维度即第一行 array[0][0]访问第二个维度

 指针和数组

  指针的定义

   指针通常用于保存地址

   指针可以是变量，可以是常量

   指针是地址的容器，地址绝对是常量

   常量指针与指针常量

    常量指针

     是一个指针，指向了一个常量，不能修改地址内的数据

     const int *ptr=&num；

     int const*ptr=&num；

    指针常量

     是一个常量，指向了一个地址，指针本身不能修改，

     int *const ptr=&num;

    指针常量指向常量

     const int *const ptr=&num；

  指针的类型

   指向的类型*指针的名字=对应数据的地址

   指针指向的类型必须和指针本身的类型相同

   不同类型的指针之间的区别主要在于指针进行算术运算时，指针变化的字节数不相同

  指针的算术运算

   指针进行加法运算 自增1相当于自增sizeof(指针类型)

     int int_number = 0; int* int_ptr = &int_number; double double_number = 0; double* double_ptr = &double_number; printf("int_ptr = 0x%p\n", int_ptr); printf("double_ptr = 0x%p\n", double_ptr); printf("int_ptr = 0x%p\n", ++int_ptr); printf("double_ptr = 0x%p\n", ++double_ptr);

   通常情况下减法没有意义，当使用数组的尾元素地址减数组的首元素地址时，得到的是两个元素直接相隔的元素个数

    即结果相当于：（地址A-地址B）/sizeof(指针类型)

     int array[10] = { 0 }; printf("&array[0] = 0x%p\n", &array[0]); printf("&array[9] = 0x%p\n", &array[9]); printf("&array[9] - &array[0] = %d\n", &array[9] - &array[0]);

  数组指针与指针数组

   数组指针

    一维数组的一些属性 int arr[10]={0}; //直接访问数组名称访问到的实际上是数组元素首地址 arr==&arr[0] arr实际上是一个数组元素的地址，需要使用int*类型的指针接收 int *arr_ptr=arr;//或int *arr_ptr=&arr[0]; 此时arr_ptr自增1，实际上arr_ptr自增了sizeof(元素数据类型) //数组本身的地址，这里数组本身的地址就是数组首元素的地址，两个值是相同的 &arr 需要保存该地址需要用到数组指针 int(*ptr)[10]=&arr 但是此时ptr自增，增加的是一个数组的大小 即sizef(arr);

    指向二维数组的指针 arr是二维数组的首地址，其值等于二维数组第一个元素（即第一个一维数组）的地址 int arr[10][10]={0}; 此时arr==&arr[0] 则接收该数组地址的指针应该是一个数组指针， int (*ptr)[10]=arr;//这里就是行指针 对于列值针，因为直接指向数组元素，所以应该 int* p=*arr 或int *p=arr[0] //相当于将第一个元素的地址赋值给ptr 第一个元素的地址值与arr是相同的，区别在于在进行算术运算时指针增加的字节数不同 ptr自增加上的是一个一维数组的长度 p自增增加的是一个数组元素的长度

 堆的使用

  堆的申请函数

   malloc：在堆空间中申请一块内存，单位是字节数

    int*p=(int*)malloc(10*sizeof(int));

   calloc:申请内存并初始化为0

    int*p=(int*)calloc(10,sizeof(int));

     申请10个int类型大小的空间

   realloc：如果允许，在原有地址上重新分配一块空间

    str = (char*)realloc(str, 20);

  堆的释放函数

   free();

   如果堆被破坏了，则会释放错误

    0xfdfdfdfd是堆的结束标志，如果被覆盖，在堆的释放过程中会导致程序崩溃

    同一个程序中，任何一块堆空间出现问题，很有可能所有其他堆空间出问题，因为堆之间是存在联系的

 生存周期和作用域

  局部变量(栈)

   定义：被定义在花括号内的变量

   作用域从定义的位置开始到所在花括号的结尾

   当不同作用域出现相同变量名称，那么小作用域的变量会覆盖大作用域的变量。

   生存周期从定义的位置开始到所在花括号的结尾

   未初始化的局部变量通常保存的是0xccccc

  全局变量（静态数据区）

   定义：定义在所有{}外的变量

   全局变量的初始化位于main函数之前

   所有全局变量会被默认初始化为0

   作用域：从定义开始到整个文件结束

    可以使用extern声明使用其他文件的全局变量

   生存周期：从程序运行开始到程序运行结束

  静态变量（静态数据区）

   定义

    使用static定义

    默认初始化为0

    无论在哪里，只被初始化1次

   生存周期

    程序运行开始到程序运行结束

   作用域

    和所定义的位置相关，如果全局那就是全局，否则就是局部

    使用static定义的全局变量不能被其他文件使用

  堆变量

   定义：编写程序的人维护的一块内存空间

   生存周期 从malloc等函数到free

   作用域：取决于定义在哪里，全局或局部

  顺序结构

  选择结构

   特点：有多条语句可以执行，但是只能执行其中部分

   if elseif else

    通常用于判断是否在某一个范围内

    推荐给每一个判断后面的语句加{}

    每一个判断语句后面只能跟一条语句或一个语句块

    如果满足前面的条件就直接执行，如果全都不满足就执行else

   switch case

    case后面跟的必须是整形（包括字符）常量

    case后面不能直接定义变量，会报错，若想在case后面定义变量，则需要加上{}

    case的值不能重复，会直接报错

    case default没有固定的顺序

    default不是必须的，如果所有case均不满足条件，就执行

    break 用于结束一个case，

    如果没有在case后面加上对应的break，那么就会接续执行下一个case
